nginx使用小记

Posted on 2020-06-19

知识网结构图

概念

正向代理：局域网中的电脑用户想要直接访问网络是不可行的，只能通过代理服务器来访问，这种代理服务就被称为正向代理。

正向代理类似一个跳板机，代理访问外部资源

比如我们国内访问谷歌，直接访问访问不到，我们可以通过一个正向代理服务器，请求发到代理服，代理服务器能够访问谷歌，这样由代理去谷歌取到返回数据，再返回给我们，这样我们就能访问谷歌了

正向代理的用途：

（1）访问原来无法访问的资源，如google
（2）可以做缓存，加速访问资源
（3）对客户端访问授权，上网进行认证
（4）代理可以记录用户访问记录（上网行为管理），对外隐藏用户信息

反向代理： 以代理服务器来接受internet上的连接请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给internet上请求连接的客户端，此时代理服务器对外就表现为一个服务器


反向代理作用

(1)保证内网的安全，组织web攻击，大型网站通常将反向代理作为公网访问地址，web服务器是内网
(2)负载均衡，通过反向代理服务器来优化网站的负载

总结

正向代理：客服端和代理对服务端透明；(proxy和客户端合伙)
反向代理：代理和服务端对客户端透明；(proxy和服务端合伙)

负载均衡

参考：https://www.cnblogs.com/muhy/p/10528449.html

在http节点下，添加upstream节点。
upstream linuxidc { //负载均衡
      server 10.0.6.108:7080; 
      server 10.0.0.85:8980; 
}
将server节点下的location节点中的proxy_pass配置为：http:// + upstream名称，即“http://linuxidc”.
location / { 
            root  html; 
            index  index.html index.htm; 
            proxy_pass http://linuxidc; 
}

现在负载均衡初步完成了。upstream默认按照轮询方式进行负载；但是还有其他的策略，包括权重、hash、第三方等方式具体见如上链接

反向代理之 proxy_pass
参考：https://www.cnblogs.com/yyxianren/p/10831511.html

格式很简单： proxy_pass URL;
其中URL包含：传输协议（http://, https://等）、主机名（域名或者IP:PORT）、uri。

示例如下：

proxy_pass http://www.xxx.com/;
proxy_pass http://192.168.200.101:8080/uri;
proxy_pass unix:/tmp/www.sock;

配置示例：

location /aming/
{
    proxy_pass http://192.168.1.10/linux/;
    ...
}

【总结】为了方便记忆和规范配置，建议所有的 proxy_pass 后的url都以“/”结尾。
proxy_pass http://192.168.1.10/linux/;

robotframework目录文件结构

robotframework是以project为单位进行管理的  
一个project可以包含多个Test Suite
一个Test Suite可以包含多个测试用例
一个Test Suite有四部分组成：Settings、Variables、Test Cases、Keywords

查看报告文件

用例执行完毕后会生成三个文件分别是：log.html、output.xml、report.html

output.xml：记录的测试结果是XML文件，根据特定的需要可以编写脚本读取XML文件并生成特定的测试报告
log.html：会记录Robotframework运行的每一步操作，主要用于编写测试脚本的过程查看
report,html：为测试报告，整理性的展示测试用例的运行情况
